## 推奨ルート

投資対効果の観点では、**「評価基盤(B-1)を土台に据えてから「チャンキング戦略(A-1)」に着手する**
のが王道。「測れる仕組み → 効く改善」の順で進めることで、精度改善の効果を定量的に確認できる。
体感価値で攻めるなら **マルチターン会話(B-2)** が分かりやすい一手。

---

## A. RAG 精度改善(検索・回答の質)

### A-1. チャンキング戦略の見直し(最有力)
現在は Fixed-size chunking。Bedrock KB は Semantic / Hierarchical chunking をサポートする。
文書の意味境界で分割することで、検索ヒット率と回答の文脈整合性が改善しやすい。
コード変更は `lib/config.ts` 中心で比較的小さい。

### A-2. 埋め込みモデルの再評価
Titan Text Embeddings V2 と Cohere multilingual 等の日本語性能を比較。
次元数とのトレードオフ(精度・コスト・ストレージ)も含めて検証する。

### A-3. メタデータフィルタリング
Drive のフォルダ階層・ファイル種別・更新日をメタデータとして付与し、検索時に絞り込む。
「最新版だけ」「特定部署フォルダだけ」といった検索が可能になる。

### A-4. クエリリライト / マルチクエリ
質問を LLM で言い換え・分解してから検索し、再現率を上げる。

### A-5. リランクと取得件数のチューニング
導入済みの Cohere Rerank について、`numberOfResults` と最終採用件数の最適点を実データで詰める。

## B. 機能追加(利用パターンの拡張)

### B-1. 評価基盤(RAG eval)(土台として最有力)
質問・正解セットで検索精度/回答品質を自動測定する。これが無いと A の改善が
「良くなった気がする」で終わる。精度改善を本気でやるなら最初に置きたい土台。

### B-2. マルチターン会話(宣言済み将来スコープ)
Slack スレッドの文脈を引き継ぐ。体感価値が大きい。

### B-3. フィードバック収集ループ
Slack で 👍 / 👎 を集計し、評価基盤・改善の燃料にする。

### B-4. 他チャネルのコンシューマ
Teams bot / 簡易 Web チャット UI など。二層構成の「利用パターン」追加の規約
(`lib/<名前>-stack.ts` + `lambda/<名前>/`)に乗せる。

### B-5. AgentCore / Agents 連携(宣言済み将来スコープ)
ツール実行・複数ステップ化。

## C. 運用・基盤・セキュリティ

### C-1. HeadObject N+1 のマニフェスト方式化
既知課題の解消。同期状態(キー → `modifiedTime`)を 1 つのマニフェスト JSON として
S3 に置き、毎回 1 回の `GetObject` で読む方式へ。数千ファイル規模での同期タイムアウトを回避。
詳細は README「既知の課題」を参照。

### C-2. アクセス制御 / 権限反映フィルタ
Drive の閲覧権限をユーザーごとに検索結果へ反映する(見えない文書を返さない)。
社内利用では重要度が高い。

### C-3. 監視・コスト可視化
CloudWatch ダッシュボード・アラート、ingestion 失敗通知など。

## D. 要約・俯瞰系の回答

「特定の文書をピンポイントで引く」現在の RAG とは別に、「フォルダ全体を要約して」
「このテーマの概要を教えて」といった俯瞰的な問いに答えられるようにする方向。
通常の RetrieveAndGenerate は取得チャンク数に上限があり、文書群全体の要約には向かない
ため、別アプローチが必要になる。

### D-1. 要約・概要回答モード(関心領域)
Map-reduce / 階層要約で「文書ごとの要約 → 統合要約」を生成する。
あるいは文書単位の事前要約をメタデータや別インデックスに保持しておき、概要質問に充てる。

### D-2. クエリルーティング(概要質問 / ピンポイント質問の振り分け)
質問の種類を判定し、「ピンポイント検索」と「俯瞰・要約」で処理経路を切り替える。
D-1 を活かす前提となる土台。

### D-3. ドキュメントカタログ / 目次の自動生成
Drive 配下にどんな文書があるかの一覧と短い要約を提示する。「何が入っているか」を俯瞰できる。

### D-4. 定期サマリ投稿
フォルダの新着・更新を定期的に要約し、Slack 等へ自動投稿する。

## E. その他の候補

- **多言語対応** — 英語ドキュメント・英語質問への対応(埋め込み・生成プロンプト両面)。
- **マルチモーダル / 図表対応** — 画像・図表を含む文書の OCR・図表理解。
- **表データへの構造化 QA** — スプレッドシート / CSV を対象にした集計・参照質問。
- **出典のディープリンク強化** — ページ番号・セクション単位でのリンク付与。
- **同期のニアリアルタイム化** — 現在のスケジュール起動に加え、Drive の変更通知を契機にした同期。
- **コスト最適化** — 応答キャッシュ、モデルの階層化(軽量モデルで一次応答)など。
